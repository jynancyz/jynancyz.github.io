Spotify Redesign

s

An project done in Cogs 187a class focusing on redesigning a music player.

Goal: go through design process of interviewing users, defining the prominent issue(s), ideating solutions, and creating a redesign.

What I did

♦   Conducted user-experience interviews with users of music players (namely Spotify, Soundcloud, & Apple Music)
♦   Analyzed and categorized data in spreadsheets both qulitatively and quantitatively
♦   Defined certain problematic features within the Spotify mobile application
♦   Ideated redesigns based on two main problems in creating and managing playlists in the Spotify mobile app


&


What I learned

♦   Comparing interfaces from other similar applications was crucial to a successful redesign as it provides certain constraints.
♦   Paper prototyping permits for ideations (a tactic of quick prototyping).
♦   The user-experience interviews revealed some expectations that users already had for the interface.

&

♦ How to use screen recording application to record performed tasks by interviewees.
♦ How to utilize quick prototyping to test different redesigns.
♦ How to utilize wireframing applications for a mobile application.
♦ How to create workflows for certain actions and sitemaps for different applications.


The Process:


1. Assignment Required Understanding Existing + Ideating Redesign

The class was split into teams of four students. The expected process included interviewing, data analysis, prototyping, and ideating and creating a redesign.



2. Current User-Experiences Noted in In-Person Interviews


All interviews were conducted in person with Master-Apprentice method. We asked for permission to have screen recordings of each user completing a list of actions which included creating a playlist and modifying it by adding and deleting songs.



3. Analysis and Categorization of Interview Data


Data is categorized and analyzed on a spreadsheet to better understand each userâ€™s experience as well as the overall issues that exist. Screen recordings were taken of the users during the interview/task completion process. However, these were associated with Lookback.io and are expired/cannot be viewed without pay. We found that many users complained about not being able to better personalize their playlists due to the amount of time it takes to add songs.



4. Definition of Most Common Problems With Comparisons


Other popular music applications were researched to better understand the features of Spotify. There were certain features that ran much more smoothly on other applications. Workflow graphs were completed to visually compare the functions of each application.These features later inspired our redesigned features.


5. Redesign Prototypes on Paper


In the image above, we ideated on the process of curating a playlist. At the moment, Spotify allows for "Discover Weekly" which provides users a playlist of songs that are similar to what they listen to. Our redesign would allow users to form their own mix based on existing mixes (including Discover Weekly).

We also ideated ways on allowing for the addition and deletion of multiple songs at once. This was a prominent complaint from our interviews about the music player. Inspiration was drawn from the multiple selection function from other applications/"Select applicable" type lists.



6. High-Fidelity Renders Created


The mockup was created with an iPhone 7 template and retains all Spotify features that are currently on the application- the multiple selection option is an addition to accomodate the need for mass deleting and adding.
